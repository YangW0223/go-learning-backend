# 每周固定节奏模板（8-10 小时）

## 时间分配

- 理论学习：2 小时
- 项目编码：4 小时
- 测试与复盘：2 小时
- 总结与文档：1-2 小时

## 建议排期（可按个人日程调整）

- 周一（30 分钟）：回顾本周目标，拆解任务，建立 issue/checklist。
- 周二（1-1.5 小时）：理论学习 + 关键概念笔记。
- 周三（1.5-2 小时）：实现核心功能第一版。
- 周四（1.5-2 小时）：补齐边界处理 + 重构。
- 周五（1 小时）：编写和完善测试。
- 周末（1.5-2 小时）：文档总结 + 演示录屏或复盘记录。

## 每周固定输出模板

- 本周完成：
- 遇到问题：
- 解决方案：
- 新学到的概念：
- 下周风险：
- 下周第一优先级：

## 每周固定检查命令

```bash
make fmt
make test
make build
```
