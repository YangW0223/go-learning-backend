# Week 08：缓存与性能

## 本周目标

为 Todo 列表引入缓存与失效策略，并理解 hit/miss 对延迟的影响。

## 详细步骤

1. 学习 Redis 基础操作与 TTL 概念。
2. 选定缓存键设计（如按用户维度或分页维度）。
3. 实现列表读取流程：先查缓存，miss 再查 DB 并回填缓存。
4. 实现写操作后的失效策略（删除或更新相关缓存键）。
5. 统计并记录 cache hit/miss。
6. 用简单压测或多次请求对比引入缓存前后的耗时。

## 建议实践清单

- 缓存数据结构保持简单（JSON 字符串或哈希）。
- 设置合理 TTL，避免陈旧数据长期存在。
- 对缓存不可用场景做降级（不影响主流程）。

## 验收清单

- [ ] Todo 列表已接入缓存。
- [ ] 有明确失效策略且已验证。
- [ ] 能解释 hit/miss 与延迟关系。
- [ ] 缓存异常不导致服务不可用。

## 产出物

- Redis 接入代码与配置。
- 命中率和延迟对比记录。
- 缓存策略说明文档。

## 常见风险与排查

- 只加缓存不做失效：数据一致性会快速恶化。
- 缓存雪崩：给 TTL 加随机抖动。
