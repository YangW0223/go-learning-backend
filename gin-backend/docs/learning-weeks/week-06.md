# Week 6 每日拆分（Day 1 ~ Day 7）

### Day 1：测试策略设计

- 今日目标：确定优先补测模块与范围。
- 操作清单：
  - 列出高风险业务路径。
  - 明确单测与接口测试分工。
  - 确定断言风格。
- 完成标准：
  - 有测试任务拆分清单。

### Day 2：service happy path

- 今日目标：补齐 service 成功路径测试。
- 操作清单：
  - mock repository。
  - 覆盖核心业务流程。
  - 验证返回值与调用次数。
- 完成标准：
  - 关键 service happy path 全通过。

### Day 3：service error path

- 今日目标：补齐 service 异常路径测试。
- 操作清单：
  - 覆盖 DB 错误、参数异常、业务冲突。
  - 校验错误码和错误文案。
  - 清理重复测试代码。
- 完成标准：
  - error path 覆盖核心失败场景。

### Day 4：handler 接口测试

- 今日目标：补 handler 层请求/响应测试。
- 操作清单：
  - 覆盖参数绑定失败、鉴权失败、成功响应。
  - 验证 HTTP 状态码与响应体格式。
  - 检查中间件行为。
- 完成标准：
  - handler 测试可覆盖关键入口。

### Day 5：测试整合与稳定性

- 今日目标：让测试稳定可重复执行。
- 操作清单：
  - 统一测试数据构造。
  - 统一命名与目录结构。
  - 执行 `go test ./...`。
- 完成标准：
  - 本地重复执行测试结果稳定。

### Day 6：补漏与优化

- 今日目标：补齐遗漏场景并提升可读性。
- 操作清单：
  - 补 2~3 个关键边界测试。
  - 提炼公共测试 helper。
  - 修复 flaky 测试。
- 完成标准：
  - 测试代码更简洁且可维护。

### Day 7：测试周复盘

- 今日目标：固化测试规范。
- 操作清单：
  - 记录测试覆盖薄弱点。
  - 记录未来新增功能测试模板。
  - 准备 Week 7 故障演练方案。
- 完成标准：
  - 输出一份测试实践说明。
