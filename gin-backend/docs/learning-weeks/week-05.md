# Week 5 每日拆分（Day 1 ~ Day 7）

### Day 1：错误现状盘点

- 今日目标：摸清全项目错误返回形态。
- 操作清单：
  - 抽样查看核心接口错误响应。
  - 统计错误码与文案不一致点。
  - 记录优先修复清单。
- 完成标准：
  - 有一份错误问题列表。

### Day 2：统一错误模型

- 今日目标：统一错误结构与映射规则。
- 操作清单：
  - 梳理 `AppError` 与 response 层。
  - 规范状态码映射。
  - 统一字段命名。
- 完成标准：
  - 主要接口错误结构一致。

### Day 3：配置治理

- 今日目标：整理并固化配置项规范。
- 操作清单：
  - 盘点环境变量、默认值、必填项。
  - 区分 dev/docker/prod 配置差异。
  - 补充缺失说明。
- 完成标准：
  - 有配置说明清单文档。

### Day 4：校验与容错

- 今日目标：提高启动阶段配置健壮性。
- 操作清单：
  - 增加关键配置校验。
  - 明确错误提示。
  - 验证错误配置下服务行为。
- 完成标准：
  - 配置错误时可快速定位问题。

### Day 5：回归测试

- 今日目标：验证错误治理未破坏业务流程。
- 操作清单：
  - 运行 `go test ./...`。
  - 回归认证和 Todo 关键接口。
  - 校验错误返回稳定性。
- 完成标准：
  - 错误处理变更平稳上线。

### Day 6：文档与规范固化

- 今日目标：团队可直接复用错误/配置规范。
- 操作清单：
  - 产出错误码对照表。
  - 产出配置字段说明表。
  - 更新 README 或 docs 索引。
- 完成标准：
  - 新成员可按文档快速上手。

### Day 7：周复盘

- 今日目标：总结治理收益与后续改进点。
- 操作清单：
  - 记录治理前后差异。
  - 记录尚未处理的问题。
  - 制定 Week 6 测试重点。
- 完成标准：
  - 输出可执行下周测试计划。
